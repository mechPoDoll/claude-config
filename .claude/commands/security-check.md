# 보안 체크리스트 실행 커맨드

$ARGUMENTS 대상 코드에 대해 보안 체크리스트를 실행합니다.
대상을 지정하지 않으면 현재 프로젝트 전체를 점검합니다.

## 공통 금지 항목 점검

아래 항목 각각에 대해 코드에 존재 여부를 확인하고 결과를 보고한다:

- [ ] 사용자 입력 → SQL 문자열 직접 삽입
- [ ] 이스케이프 없는 사용자 데이터 출력
- [ ] CSRF 토큰 없는 상태 변경 요청
- [ ] 인증 체크 없는 처리 엔드포인트
- [ ] 평문 비밀번호 저장
- [ ] `.env` / 시크릿 파일 커밋 여부 확인
- [ ] 운영 환경 상세 에러 노출
- [ ] 사용자 입력 → 셸 명령 직접 전달
- [ ] 파일 경로에 사용자 입력 직접 사용 (`../` 포함)
- [ ] 불필요한 개인정보 수집/로깅

## 공통 보안 항목 점검

- [ ] HTTPS 강제 적용 여부
- [ ] 보안 헤더 설정 여부
- [ ] CORS 설정 확인 (`*` 사용 여부)
- [ ] Rate Limiting 적용 여부
- [ ] 쿠키 HttpOnly / Secure / SameSite 설정
- [ ] 약한 암호화 알고리즘 사용 여부 (MD5, SHA-1)

## 언어별 추가 점검

현재 프로젝트 언어를 감지하여 해당 항목을 추가로 점검한다:

**PHP**: PDO/Prepared Statement 사용 여부, `eval()`/`exec()` 사용 여부, `display_errors` 설정
**JavaScript**: `innerHTML`/`eval()` 직접 사용 여부, JWT 시크릿 하드코딩 여부
**Python**: `eval()` 사용 여부, `pickle.loads()` 사용 여부, Django DEBUG 설정
**Java**: PreparedStatement/JPA 사용 여부, Entity 직접 노출 여부, `${}` MyBatis 사용 여부

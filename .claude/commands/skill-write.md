# SKILL 명세서 작성 커맨드

$ARGUMENTS 기능/화면에 대한 SKILL 명세서를 작성합니다.

## 파일 명명 규칙 (반드시 준수)

| 파일명 패턴 | 용도 |
|------------|------|
| `00_OVERVIEW.md` | 프로젝트 전체 IA, 파일 매핑 (항상 최초 생성) |
| `NN_섹션명.md` | 메뉴/기능별 명세서 (NN = 메뉴번호, 두 자리) |
| `CHANGE_MMDD.md` | 기존 기능 변경 요청 명세 |
| `DB_SCHEMA.md` | DB 스키마 정의 |
| `index.md` | 마일스톤 트래커 |

## 명세서 작성 절차

1. `SKILL/00_OVERVIEW.md`가 없으면 먼저 생성한다
2. `SKILL/index.md`가 없으면 생성한다
3. 해당 기능의 `NN_섹션명.md` 파일을 작성한다
4. `SKILL/index.md`에 새 항목을 추가한다

## MUST 섹션 (반드시 포함)

모든 `NN_섹션명.md`에 아래 섹션은 정보 부족 시 `{{TODO: 확인 필요}}`로 표기하되 생략 금지:

- **페이지 개요** — 페이지명, Figma 프레임, 관련 파일, 접근 권한
- **기능 요약** — 1~2문장 설명
- **ASCII 와이어프레임** — 텍스트 기반 UI 표현

DB 연동 시 추가 MUST:
- **데이터베이스 테이블 구조**

목록 화면인 경우 추가 MUST:
- **목록 컬럼 정의** — 컬럼명, DB 필드, 비고
- **필터 조건** — 필터명, 필터ID, 옵션, DB 매핑

버튼 있는 화면인 경우 추가 MUST:
- **버튼 기능** — 버튼명, 동작

## 명세서 작성 형식

```markdown
# NN. 섹션명 (Page X)

## 페이지 개요
| 항목 | 내용 |
|------|------|
| 페이지명 | {{페이지명}} |
| Figma 프레임 | {{frameId}} |
| 관련 파일 | {{파일 경로}} |
| 접근 권한 | {{권한 조건}} |

## 기능 요약
{{1~2문장 설명}}

## ASCII 와이어프레임
{{텍스트 기반 UI 표현}}

## 데이터베이스 테이블 구조 (DB 연동 시)
{{관련 테이블 정의}}

## 화면별 상세
### N-1. 하위화면명
#### ASCII 와이어프레임
#### 필터 조건 (필터 있는 화면)
#### 목록 컬럼 정의 (목록 화면)
#### 버튼 기능 (버튼 있는 화면)
```

## index.md 갱신 형식

```markdown
## 1. 섹션명

|----|------|-------------|------|------|
| M01 | 화면명 | {{frameId}} | {{파일}} | ⬜/🔄/✅ |
```

# QA 보안 감사 커맨드

$ARGUMENTS 대상에 대해 보안 감사를 수행합니다.

## 파일 명명 규칙 (반드시 준수)

| 유형 | 파일명 패턴 | 예시 |
|------|------------|------|
| QA 이력 마스터 | `index.md` | 전체 QA 현황 요약 |
| 보안감사 보고서 | `qa_YYYYMMDD_v#.md` | `qa_20260318_v1.md` |
| 주제별 보고서 | `qa_YYYYMMDD_v#_주제.md` | `qa_20260318_v2_db_tuning.md` |
| 레거시 체크리스트 | `legacy_checklist.md` | 레거시 초기 점검용 |

## 보안 감사 절차

1. `QA/index.md`가 없으면 먼저 생성한다
2. 대상 파일/디렉터리를 스캔하여 아래 보안 항목을 점검한다
3. **Playwright 자동 검증 여부를 사용자에게 질문한다:**
   > "Playwright를 사용하여 UI/기능을 자동 검증할까요? (브라우저 테스트가 가능한 환경인 경우 권장)"
   > - Yes → Playwright로 주요 화면 접근·폼 제출·인증 흐름 등을 자동 테스트하고 결과를 보고서에 포함
   > - No → 코드 정적 분석만으로 점검 진행
4. 오늘 날짜 기준 `qa_YYYYMMDD_v1.md` 파일명으로 보고서를 작성한다
5. `QA/index.md`에 새 보고서를 추가한다

## 점검 항목 (공통)

- [ ] 사용자 입력 → SQL 문자열 직접 삽입 (SQL Injection)
- [ ] 이스케이프 없는 사용자 데이터 출력 (XSS)
- [ ] CSRF 토큰 미적용 상태 변경 요청
- [ ] 인증 체크 없는 처리 엔드포인트
- [ ] 평문 비밀번호 저장
- [ ] `.env` / 시크릿 파일 커밋 여부
- [ ] 운영 환경 상세 에러 노출
- [ ] 사용자 입력 → 셸 명령 직접 전달
- [ ] 파일 경로에 사용자 입력 직접 사용 (Path Traversal)

## 점검 항목 (FTP/SFTP 환경 추가)

- [ ] `.vscode/sftp.json`에 `.env`, `.git` 등 ignore 설정 여부
- [ ] FTP 접속 정보(호스트, 비밀번호)가 코드에 하드코딩 여부
- [ ] 배포 제외 파일이 서버에 노출되지 않는지 확인 (`.git/`, `.env`, `node_modules/`)
- [ ] 배포 전/후 파일 목록 기록 여부 (변경 추적)

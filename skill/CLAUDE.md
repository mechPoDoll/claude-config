# Skill 디렉터리 규칙

## 목적
이 디렉터리는 폴더구조 파서에서 사용되는 스킬(기능 모듈) 목록과 설명을 관리합니다.

## 스킬 문서 작성 규칙
- 각 스킬은 개별 파일로 관리하며, 파일명은 스킬 이름과 동일하게 작성
- 스킬 문서에는 **설명**, **입력**, **출력**, **사용 예시**를 반드시 포함
- 새로운 스킬 추가 시 이 디렉터리에 문서를 먼저 작성한 후 구현할 것

## 스킬 문서 템플릿
```markdown
# 스킬명

## 설명
이 스킬이 하는 일을 한 줄로 요약

## 입력
- `param1` (타입): 설명
- `param2` (타입): 설명

## 출력
- 반환값 설명

## 사용 예시
(코드 또는 CLI 예시)
```

## 스킬 분류
- **파싱**: 폴더/파일 구조를 읽고 트리로 변환
- **생성**: 파싱 결과를 기반으로 문서 파일 생성
- **필터링**: 특정 패턴의 파일/폴더를 포함하거나 제외
- **포맷팅**: 출력 형식 변환 (Markdown, JSON, YAML 등)

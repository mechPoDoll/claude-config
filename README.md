# claude-config — CLAUDE.md 설정 관리

Claude Code에서 사용하는 CLAUDE.md 파일을 계층별로 관리하고, 새 프로젝트에 복사하여 바로 사용할 수 있도록 구성한 레포지터리입니다.

---

## 이 레포가 하는 일

1. **글로벌 규칙**을 버전 관리합니다 (`GLOBAL/CLAUDE.md` → `~/.claude/CLAUDE.md`에 복사하여 적용)
2. **프로젝트 루트 CLAUDE.md**를 새 프로젝트에 넣으면, 소스 코드를 자동 분석하여 `{{플레이스홀더}}`를 채워줍니다
3. **QA/SKILL 디렉터리**를 프로젝트에 넣으면, 작업 진행에 따라 보고서·명세서가 규칙대로 생성됩니다

---

## 디렉터리 구조

```
claude-config/
├── CLAUDE.md              # 프로젝트 루트 규칙 (자동 분석 지시문 포함)
├── GLOBAL/
│   └── CLAUDE.md          # 글로벌 규칙 (~/.claude/CLAUDE.md 원본)
├── QA/
│   └── CLAUDE.md          # QA 디렉터리 규칙
├── SKILL/
│   └── CLAUDE.md          # Skill 디렉터리 규칙 + 기능별 명세서
└── README.md              # 이 파일
```

---

## 사용 방법

### 1. 글로벌 설정 적용

```bash
cp GLOBAL/CLAUDE.md ~/.claude/CLAUDE.md
```

이후 모든 프로젝트에서 보안 체크리스트, 금지 사항, 상태 아이콘, Figma 행동 원칙이 자동 적용됩니다.

### 2. 새 프로젝트에 적용

```bash
# 대상 프로젝트 루트에 파일 복사
cp CLAUDE.md /path/to/project/CLAUDE.md
cp -r QA /path/to/project/QA
cp -r SKILL /path/to/project/SKILL
```

프로젝트에서 Claude Code를 시작하면:
- `CLAUDE.md`의 `{{플레이스홀더}}`를 감지
- 소스 코드를 자동 스캔 (package.json, composer.json, requirements.txt 등)
- 분석 결과로 기술 스택, 명령어, 개발 규칙, 보안 패턴 등을 채움
- 사용자 확인 후 저장

### 3. 기존 프로젝트에서 지속 업데이트

이미 채워진 CLAUDE.md가 있는 프로젝트에서는:
- 플레이스홀더가 없는 섹션은 건드리지 않음
- QA/SKILL 디렉터리에 작업 산출물이 규칙에 따라 계속 추가됨
- index.md 등 트래커 문서가 진행 상황에 따라 갱신됨

---

## CLAUDE.md 계층 구조

```
~/.claude/CLAUDE.md          ← 글로벌 (모든 프로젝트에 적용)
  └ /GLOBAL/CLAUDE.md        ← 이 레포에서 버전 관리하는 원본
/CLAUDE.md                   ← 프로젝트 루트
/QA/CLAUDE.md                ← QA 디렉터리
/SKILL/CLAUDE.md             ← Skill 디렉터리
```

하위로 갈수록 범위가 좁아지고, 상위 규칙을 상속합니다.

---

## 각 파일의 역할

### GLOBAL/CLAUDE.md — 모든 프로젝트 공통

| 항목 | 설명 |
|------|------|
| Figma 작업 워크플로우 | 디자인 → 코드 구현 시 행동 원칙 (4단계) |
| SFTP 설정 규칙 | `.vscode/sftp.json` 기본 ignore 목록 |
| 공통 상태 아이콘 | ✅ 🔄 ❌ ⬜ — QA/SKILL 전체에서 통일 사용 |
| 금지 사항 체크리스트 | SQL Injection, XSS, CSRF 등 10개 항목 |
| 보안 체크리스트 | PHP, JavaScript, Python, Java 언어별 + 공통 규칙 |
| 수정 작업 리포트 | 완성도 점수 (1~100) 보고 형식 |

### CLAUDE.md (프로젝트 루트) — 프로젝트별 설정

| 섹션 | 설명 |
|------|------|
| 자동 분석 지시문 | `{{플레이스홀더}}` 감지 시 소스 분석 → 자동 채움 |
| 프로젝트 개요 | 기술 스택, 디렉터리 구조 |
| 명령어 | 시작/중지/빌드/DB 접속 |
| 개발 규칙 | 파일 명명, Import 패턴, DB 접근, 응답 패턴 |
| 보안 규칙 | 글로벌 상속 + 프로젝트 특화 규칙만 추가 |
| 코드 패턴 | 목록 페이지, CRUD 핸들러 표준 패턴 |

> 모든 예시는 특정 언어에 종속되지 않으며, 자동 분석 시 프로젝트 언어에 맞게 채워집니다.

### QA/CLAUDE.md — 품질 보증

| 항목 | 설명 |
|------|------|
| 워크플로우 | 보안 감사 → 조치 → 테스트 → 배포 → 검증 |
| 파일 명명 규칙 | `qa_YYYYMMDD_v#.md`, `e2e-범위.*`, `deploy.*` 등 |
| 보고서 형식 | 보안감사, DB 튜닝, 기능감사 3종 템플릿 |
| 테스트 작성 지침 | 도구 무관 (Playwright, Cypress 등 프로젝트에 따라 선택) |
| 배포 스크립트 지침 | 방식 무관 (FTP, SSH, CI/CD 등 프로젝트에 따라 선택) |
| 진행도 보고 | 프로그레스 바 (█░) + 화면별 상세 + 최종 QA 판정 |

> 초기에는 CLAUDE.md만 존재하며, 진행에 따라 산출물이 생성됩니다.

### SKILL/CLAUDE.md — 기능 명세서

| 항목 | 설명 |
|------|------|
| 워크플로우 | Figma 수집 → 분석 MD 생성 → 개발 검증 → 퍼블 참고 → 점수화 |
| 파일 명명 규칙 | `00_OVERVIEW.md`, `NN_섹션명.md`, `USER_NN_섹션명.md`, `DB_SCHEMA.md` |
| 스킬 문서 형식 | 페이지 개요, 와이어프레임, DB 구조, 필터/컬럼/쿼리/버튼 상세 |
| 마일스톤 트래커 | `index.md` / `user_index.md`로 화면별 진행 상태 관리 |
| 퍼블 점수화 | 레이아웃/스타일/반응형/인터랙션/접근성 5개 항목 100점 만점 |

> 초기에는 CLAUDE.md만 존재하며, Figma 분석에 따라 명세서가 생성됩니다.

## 설계 원칙

| 원칙 | 설명 |
|------|------|
| 언어 무관 | 특정 언어/프레임워크에 종속되지 않음. 자동 분석으로 프로젝트에 맞게 적용 |
| MD 파일만으로 시작 | 각 디렉터리에 CLAUDE.md만 넣으면 동작. 추가 도구/스크립트 불필요 |
| 상위 규칙 상속 | GLOBAL → 루트 → 하위 순으로 상속. 중복 없이 각 계층에서 해당 역할만 담당 |
| 점진적 확장 | 작업 진행에 따라 보고서, 명세서, 트래커가 규칙대로 자동 생성·갱신 |
| 상충 없음 | GLOBAL에서 공통 규칙(아이콘, 금지 사항) 정의 → 하위에서 참조만 |

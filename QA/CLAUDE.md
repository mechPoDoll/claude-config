# QA 디렉터리 규칙

## 목적
이 디렉터리는 프로젝트의 품질 보증(QA) 관련 문서를 관리합니다.

## 문서 작성 규칙
- 테스트 케이스 문서는 `TC-XXX` 형식으로 번호를 부여할 것
- 버그 리포트는 **재현 순서**, **기대 결과**, **실제 결과**를 반드시 포함
- 테스트 결과는 날짜별로 정리하고, 파일명에 `YYYY-MM-DD` 형식 사용

## 테스트 범위
- 단위 테스트: 개별 파서 함수의 입출력 검증
- 통합 테스트: 폴더 구조 전체 파싱 → 문서 생성 흐름 검증
- 엣지 케이스: 빈 폴더, 심볼릭 링크, 숨김 파일, 깊은 중첩 구조

## 품질 기준
- 파서 정확도: 실제 폴더 구조와 출력 문서 간 100% 일치
- 성능: 1,000개 이하 파일/폴더 구조에서 5초 이내 처리
- 에러 처리: 접근 불가 경로에 대해 명확한 에러 메시지 출력
